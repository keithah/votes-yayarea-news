# Launch Verification Artifacts

This directory contains durable, redaction-safe launch verification evidence for final static-export gates.

## S07 public-site launch evidence

S07 is the M003 launch-safety gate for the completed public site under the static GitHub Pages boundary. The current canonical summary is `latest.json`; for S07 it mirrors `s07-launch-verification.json`.

### `s07-launch-export.json`

Generated by `scripts/assert-s07-export.mjs --json-out data/launch/s07-launch-export.json`. It validates data-derived public route contracts, internal links, source coverage, count honesty, leakage checks, and absence of local-only route artifacts across the static export.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S07"`
- `generatedBy: "scripts/assert-s07-export.mjs"`
- `status: "pass"`
- `counts` with route, link, leakage, local-only artifact, and coverage-race totals
- `checkedRoutes[]`, `coverageContracts[]`, and phase-labeled diagnostics

### `s07-static-smoke.json`

Generated by `scripts/smoke-s07-static-export.mjs --json-out data/launch/s07-static-smoke.json`. It serves `out/` over `127.0.0.1`, crawls every S07 public route contract, verifies slash redirects, samples exported assets, and records phase-labeled HTTP diagnostics.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S07"`
- `generatedBy: "scripts/smoke-s07-static-export.mjs"`
- `status: "pass"`
- `origin` as a local HTTP origin
- `checkedRoutes[]`, `redirectChecks[]`, `assetChecks[]`, `counts`, and `phases`

### `s07-pages-proof.json`

Generated by `scripts/record-s07-launch-verification.mjs`. It records the `.github/workflows/pages.yml` contract needed for GitHub Pages deployment:

- Node 24 setup
- Corepack and frozen pnpm install
- typecheck, data validation, and static build phases
- `GITHUB_PAGES=true` and `NEXT_PUBLIC_SITE_ORIGIN` during build
- removal of local-only route artifacts before upload
- `.nojekyll` marker creation
- upload of the `out` static export directory

### `s07-launch-verification.json` and `latest.json`

Generated by `scripts/record-s07-launch-verification.mjs`. The recorder validates `s07-launch-export.json`, `s07-static-smoke.json`, and the Pages workflow proof, then writes a single launch summary containing:

- schema version, slice, generator, timestamp, status, and artifact paths
- route class totals and static-smoke route totals
- registered-source coverage summaries and reviewed/unresolved count summaries
- link, leakage, local-only artifact, redirect, and asset totals
- workflow check statuses
- redaction status and gate-level error arrays

## S05 Launch Verification Artifacts

This directory also contains durable, redaction-safe launch verification evidence for the M002/S05 final static-export gate.

## `s05-static-smoke.json`

Generated by `scripts/smoke-s05-static-export.mjs`. It serves `out/` over `127.0.0.1`, samples representative static routes and assets, and records phase-labeled HTTP diagnostics.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S05"`
- `generatedBy: "scripts/smoke-s05-static-export.mjs"`
- `status: "pass"`
- `startedAt` and `completedAt` ISO timestamps
- `origin` as a local HTTP origin
- `checkedRoutes[]`, `redirectChecks[]`, `assetChecks[]`, `counts`, and `phases`

## `s05-browser-checks.json`

Recorded with browser tooling against a locally served `out/` site. This is representative browser evidence, not an exhaustive crawl; broad route/link/leak coverage is handled by `s05-launch-export.json` and `s05-static-smoke.json`.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S05"`
- `generatedBy` identifying the browser tooling used
- `checkedAt` ISO timestamp
- `origin` as `http://127.0.0.1:<port>` or `http://localhost:<port>`
- `devices[]` with at least one `desktop` and one `mobile` entry

Each `devices[]` entry must include:

- `name` and `kind` (`desktop` or `mobile`)
- `viewport.width`, `viewport.height`, and optional `viewport.isMobile`
- `userAgent` when available from the browser context
- `consoleErrors: []`; any console error is launch-blocking
- `routes[]` covering route classes `homepage`, `race`, `source`, `entity`, and `disclosure`

Each browser route entry must include:

- `route` as a root-relative route
- `className` (`homepage`, `race`, `source`, `entity`, `disclosure`, or an optional extra class such as `office` or `measure`)
- `url` starting with the recorded `origin`
- `status: 200`
- optional `title`
- `assertions[]`, where every assertion has `name` and `status: "pass"`

Required representative browser routes include `/`, `/races/california-governor/`, one generated source route, one generated entity route, and `/how-we-use-ai/`. Optional San Francisco office or measure routes may be included when present in the export.

## M004/S05 live GitHub Pages verification artifacts

M004/S05 is the R026 launch gate for the repaired public comparison aggregation. Passing local export checks is not sufficient for this gate: `pnpm verify:m004:s05` must also fetch the live GitHub Pages site and prove representative M004 race pages expose expanded multi-source comparison coverage.

Run the verifier with the production Pages origin by default:

```sh
pnpm verify:m004:s05
```

To verify a staging or preview deployment, override only the live fetch origin:

```sh
M004_S05_LIVE_ORIGIN=https://example.test/votes-yayarea-news pnpm verify:m004:s05
```

The command runs phase-labeled gates in order:

1. `pnpm verify:m004:s04` refreshes the local S04 static-export evidence and stops before live checks if the repaired comparison model regresses.
2. A GitHub Pages-mode `pnpm build` runs with repository defaults unless already set in the environment: `GITHUB_PAGES=true`, `GITHUB_REPOSITORY=keithah/votes-yayarea-news`, and `NEXT_PUBLIC_SITE_ORIGIN=https://keithah.github.io/votes-yayarea-news`.
3. `scripts/assert-m004-s04-public-routes.mjs` rechecks the Pages-mode `out/` export.
4. `scripts/assert-m004-s05-live-pages.mjs` fetches the live representative race pages and writes `m004-s05-live-pages.json`.
5. `scripts/record-m004-s05-launch-verification.mjs` validates the live report and writes `m004-s05-launch-verification.json` plus `latest.json` only after every gate passes.

### `m004-s05-live-pages.json`

Generated by `scripts/assert-m004-s05-live-pages.mjs --json-out data/launch/m004-s05-live-pages.json`. It records the live origin, checked routes, HTTP status, response bytes, marker assertions, leakage findings, timestamps, and phase-labeled diagnostics for:

- `/races/state-assembly-district-17/`
- `/races/california-governor/`
- `/races/us-house-district-11/`

The report must have `milestone: "M004"`, `slice: "S05"`, `status: "pass"`, three representative routes, HTTP 200 for every route, positive byte counts, all marker assertions passing, and no leakage findings.

### `m004-s05-launch-verification.json` and `latest.json`

Generated by `scripts/record-m004-s05-launch-verification.mjs`. The recorder validates `m004-s05-live-pages.json`, rejects stale slice IDs, non-pass reports, missing route coverage, and private leakage, then writes a compact launch summary with artifact paths, live-origin summary, route totals, marker totals, byte totals, redaction status, and gate errors. `latest.json` is updated only after this final report validates.

## Redaction boundary

Committed launch artifacts must not include `.gsd`, `/home/`, `file://`, secrets, or private manual-review paths. Validators also reject serialized public-route leaks for local-only debug/review URLs while allowing bounded diagnostic labels that prove those artifacts were removed from `out/` before upload. M004/S05 live artifacts intentionally omit full HTML bodies and private path match text; they retain only route names, status codes, byte counts, marker names, and leakage pattern identifiers needed to debug stale deployments.
