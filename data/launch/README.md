# S05 Launch Verification Artifacts

This directory contains durable, redaction-safe launch verification evidence for the M002/S05 final static-export gate.

## `s05-static-smoke.json`

Generated by `scripts/smoke-s05-static-export.mjs`. It serves `out/` over `127.0.0.1`, samples representative static routes and assets, and records phase-labeled HTTP diagnostics.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S05"`
- `generatedBy: "scripts/smoke-s05-static-export.mjs"`
- `status: "pass"`
- `startedAt` and `completedAt` ISO timestamps
- `origin` as a local HTTP origin
- `checkedRoutes[]`, `redirectChecks[]`, `assetChecks[]`, `counts`, and `phases`

## `s05-browser-checks.json`

Recorded with browser tooling against a locally served `out/` site. This is representative browser evidence, not an exhaustive crawl; broad route/link/leak coverage is handled by `s05-launch-export.json` and `s05-static-smoke.json`.

Required top-level fields:

- `schemaVersion: 1`
- `slice: "S05"`
- `generatedBy` identifying the browser tooling used
- `checkedAt` ISO timestamp
- `origin` as `http://127.0.0.1:<port>` or `http://localhost:<port>`
- `devices[]` with at least one `desktop` and one `mobile` entry

Each `devices[]` entry must include:

- `name` and `kind` (`desktop` or `mobile`)
- `viewport.width`, `viewport.height`, and optional `viewport.isMobile`
- `userAgent` when available from the browser context
- `consoleErrors: []`; any console error is launch-blocking
- `routes[]` covering route classes `homepage`, `race`, `source`, `entity`, and `disclosure`

Each browser route entry must include:

- `route` as a root-relative route
- `className` (`homepage`, `race`, `source`, `entity`, `disclosure`, or an optional extra class such as `office` or `measure`)
- `url` starting with the recorded `origin`
- `status: 200`
- optional `title`
- `assertions[]`, where every assertion has `name` and `status: "pass"`

Required representative browser routes include `/`, `/races/california-governor/`, one generated source route, one generated entity route, and `/how-we-use-ai/`. Optional San Francisco office or measure routes may be included when present in the export.

## `s05-launch-export.json`

Generated by `scripts/assert-s05-launch-export.mjs`. It validates route contracts, internal links, and public-trust leakage across the static export.

## `s05-launch-verification.json`

Generated by `scripts/record-s05-launch-verification.mjs`. It validates and summarizes the three launch gates above:

- `staticSmoke`
- `browserEvidence`
- `routeLinkLeakage`

## Redaction boundary

Committed launch artifacts must not include `.gsd`, `/home/`, `file://`, secrets, or private manual-review paths. Validators reject those values in serialized reports.
