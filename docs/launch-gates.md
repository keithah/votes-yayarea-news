# S09 launch gates

S09 records the public launch-readiness surface in `data/launch/latest.json`. The file is generated by:

```bash
pnpm verify:s09
```

The command is local, credential-free, and network-free. It validates public data, runs data/share/analytics tests, typechecks, builds the static export, asserts representative exported HTML, serves `out/` through a production-like static HTTP server, and records launch-gate state.

## Gate interpretation

`data/launch/latest.json` has `overallStatus` plus detailed `gates`:

- `routes` — representative exported routes exist for `/`, `/races/mayor/`, `/how-we-use-ai/`, `/entities/sample-candidate-a/`, and `/sources/san-francisco-chronicle-editorial-board/`.
- `metadataShareStatus` — each route has a title, canonical URL, Open Graph image, and Twitter image.
- `analyticsEventCoverage` — required anonymous static-compatible event markers are present: `race_page_view`, `recommendation_matrix_open`, `ai_summary_expand`, and `receipt_drawer_open`.
- `publicTrustLeakChecks` — exported HTML avoids private planning paths, review staging paths, hidden extraction drafts, machine-local paths, file URLs, and directive endorsement language.
- `staticSmoke` — a local static server returns 200 HTML for representative routes and redirects extensionless no-slash paths to the canonical trailing-slash paths.
- `manualLighthouseBrowserNotes` — explicit manual/browser notes. These may remain `pending` in local closeout, but must be reviewed before making external public-launch claims.

`overallStatus: "pass"` means all automated blocking gates passed. Manual notes are intentionally explicit because Lighthouse runs, share-card previews in external crawlers, and final browser review depend on the deployment surface rather than repository-only state.

## Failure triage

Each S09 script prefixes failures with a phase label:

- `export-files` or `route-html` — rerun `pnpm build`, then inspect the named `out/` route.
- `share-metadata` — inspect the route `generateMetadata` implementation and `lib/share/metadata.ts`.
- `analytics-coverage` — inspect `lib/analytics/events.ts` and `data-analytics-event` markers in race-page client components.
- `public-trust-leaks` — remove private paths, hidden review/draft references, local absolute paths, file URLs, or directive endorsement language from public HTML.
- `server-start`, `route-fetch`, `content-type`, or `trailing-slash` — inspect the static export smoke server output and `next.config.mjs` static export settings.
- `launch-gate-json` or `launch-gate-status` — inspect `scripts/record-s09-launch-gates.mjs` and `data/launch/latest.json` for malformed or failing gate data.
